搭建个人Blog的一点建议

  |   更新于 2017-05-28 发表于 2017-05-27 | 分类于 搭站 | 阅读次数

   博客是从27号下午开始着手搭建，一直做到28号晚上，耗时15h，加上和峰兄讨论，直到凌晨2点才觉得心里比较踏实。

   所以，感觉初次搭建个人博客的还是没有想象的那般轻松。总结出经验是：保持头脑绝对清晰，注意每一个小细节。不然，很容易范一些很二的错误，比如编辑config文件时冒号后面少了空格，或是添加CNAME时漏了www.前缀，都是很致命的。

   希望写下这篇博客，帮助更多想搭建Blog的朋友绕开一些坑。


前期准备：

   介绍我自己电脑的搭站环境是：

Git

Node.js

Hexo

创建个人GitHub仓库：

   登录到GitHub，若你没有账户，用邮箱注册即可 点击GitHub中的New repository， 这个就是你的个人仓库了，注意仓库名应该和你的GitHub用户名一样 如我的仓库名称 SuperPeterRui.github.io 我的账号名称也是 SuperPeterRui。

  • 初识MarkDown语法

获得个人网站域名:

  域名是网站的入口，也是网站的第一印象，比如饿了么的官网的域名是：https://www.ele.me/ ，很是巧妙。常见的有com,cn,net,org等后缀，也有小众的xyz,me,io等后缀，根据你自己的喜好，选择不同的后缀，比如我选择就是常见的com后缀。很多小众奇特的后缀在大陆是没办法备案的，网站也就无法上线。然而使用GitHub托管我们的网站，完全不需要备案，因为托管我们的网站内容的服务器在美国，而且在国内备案流程也比较繁杂，时间需要一周左右。

申请域名的地方有很多，这里推荐阿里云：阿里云-为了无法计算的价值 申请入口：域名注册 购买域名这也是我们整个搭建过程中惟一一个需要花钱的地方。大概45RMB就可以申请一年的域名。

安装Git

   什么是Git ?

简单来说Git是开源的分布式版本控制系统，用于敏捷高效地处理项目。我们网站在本地搭建好了，需要使用Git同步到GitHub上。

现在的机子基本都是64位的，选择64位的安装包，下载后安装，在命令行里输入git测试是否安装成功，若安装失败，参看其他详细的Git安装教程。安装成功后，将你的Git与GitHub帐号绑定，鼠标右击打开Git Bash.

   你需要在黑色的终端里输入代码：

git config --global user.name "你的GitHub用户名"

git config --global user.email "你的GitHub注册邮箱"

   生成ssh密钥文件：

   ssh-keygen -t rsa -C”你的GitHub注册邮箱”

   然后直接三个回车即可， 默认不需要设置密码

   然后再到生成的.ssh的文件夹中的id_rsa.pub密钥，用写字本的方式打开，将内容全部复制

   *之后你几乎的指令所有的操作基本都要在终端进行

   打开GitHub_Settings_keys 页面，新建new SSH Key

   Title为标题，任意填即可，将刚刚复制的id_rsa.pub内容粘贴进去，最后点击Add SSH key。
在Git Bash中检测GitHub公钥设置是否成功，输入 ssh


安装Node.js

   Hexo基于Node.js，Node.js下载地址：Download | Node.js 下载安装包，注意安装Node.js会包含环境变量及npm的安装，安装后，检测Node.js是否安装成功，在命令行中输入 node -v :

   > 若显示v6. 10. 2 则成功

   检测npm是否安装成功，在命令行输入nom -v

   > 若显示3. 10. 10 则成功


以上就完成前期搭建准备，接下来：

安装Hexo

   Hexo就是我们的个人博客网站的框架， 这里需要自己在电脑常里创建一个文件夹，可以命名为Blog，Hexo框架与以后你自己发布的网页都在这个文件夹中。创建好后，进入文件夹中，按住shift键，右击鼠标点击命令行

   输入：

  

   接着我们分别输入三条命令：

hexo new test_my_site

hexo g

hexo s

   *之后我会单独写一片文章介绍相关指令

   完成后，打开你的浏览器输入地址：

   localhost：4000

   可以看到我们的第一篇博客

推送网站

   上面只是在本地预览，接下来要做的就是就是推送网站，也就是发布网站，让我们的网站可以被更多的人访问。在设置之前，需要解释一个概念，在blog根目录里的_config.yml文件称为站点配置文件，如下图

   进入根目录里的themes文件夹，里面也有个_config.yml文件，这个称为主题配置文件，

   下一步将我们的Hexo与GitHub关联起来，打开站点的配置文件_config.yml（注意，一定要用写字本格式打开，不然格式很容易写错，hexo对格式要求非常严格），翻到最后修改为：deploy: type: gitrepo: 这里填入你之前在GitHub上创建仓库的完整路径，记得加上 .gitbranch: master参考如下：

deploy: 

  type: git

  repo: https://github.com/SuperPeterRui/SuperPeterRui.github.io.git

branch: master

   请注意每个冒号后面都有一个空格

   保存站点配置文件。

   其实就是给hexo d 这个命令做相应的配置，让hexo知道你要把blog部署在哪个位置，很显然，我们部署在我们GitHub的仓库里。最后安装Git部署插件，输入命令：

   npm install hexo-deployer-git --save

   这时，我们分别输入三条命令：

hexo clean 

hexo g 

hexo d

   其实第三条的 hexo d 就是部署网站命令，d是deploy的缩写。完成后，打开浏览器，在地址栏输入你的放置个人网站的仓库路径，即 http://xxxx.github.io

   你就会发现你的博客已经上线了，可以在网络上被访问了。

绑定域名

   虽然在Internet上可以访问我们的网站，但是网址是GitHub提供的:http://xxxx.github.io (知乎排版可能会出现”http://"字样) 而我们想使用我们自己的个性化域名，这就需要绑定我们自己的域名。这里演示的是在阿里云万网的域名绑定，在国内主流的域名代理厂商也就阿里云和腾讯云。登录到阿里云，进入管理控制台的域名列表，找到你的个性化域名，进入解析—>添加解析—>包括添加三条解析记录，192.30.255.112是GitHub的地址，你也可以ping你的 http://xxxx.github.io 的ip地址，填入进去。第三个记录类型是CNAME，CNAME的记录值是：你的用户名.http://github.io 这里千万别弄错了。第二步，登录GitHub，进入之前创建的仓库，点击settings，设置Custom domain，输入你的域名

  点击save保存。第三步，进入本地博客文件夹 ，进入blog/source目录下，创建一个记事本文件，输入你的域名，对，只要写进你自己的域名即可

  保存，命名为CNAME ，注意保存成所有文件而不是txt文件。

  完成这三步，进入blog目录中，按住shift键右击打开命令行，依次输入：

hexo clean
hexo g
hexo d

  这时候打开浏览器在地址栏输入你的个性化域名将会直接进入你自己搭建的网站。
